🔄 COMPLETE FLOW - Get Nickname Button

🎯 Step 1: Button in HTML
<button id="btnGetNickname" class="btn btn-info w-100">Get Nickname</button>
This creates a blue button labeled "Get Nickname" on the webpage.

⚠️ Step 2: JavaScript Event Handler
btnGetNickname.addEventListener('click', (e) => getNickname(e.currentTarget));
When someone presses the button, this code runs. It calls the getNickname function.

🧹 Step 3: Clear Log & Disable Buttons
async function getNickname(clickedButton) {
    clearLogAndDisableButtons('Getting nickname...', clickedButton);
    try {
        const response = await fetch('/api/aps/nickname');
        const data = await response.json();
        logMessage(`--- RESPONSE (Status: ${response.status}) ---`);
        logMessage(`<pre>${JSON.stringify(data, null, 2)}</pre>`);
This function clears the output window, shows "Getting nickname...", makes all buttons gray, then sends a GET request to the server.

📡 Step 4: Send Request to Server
The browser sends a GET request to "/api/aps/nickname" to check what nickname is currently assigned to your app.

🖥️ Step 5: Server Checks Nickname
router.get('/', async (req, res) => {
    try {
        const headers = await getHeaders();
        const response = await axios.get(`${DA_BASE_URL}/forgeapps/me`, { headers });
        res.status(200).json(response.data);
    } catch (err) {
        if (err.response && err.response.status === 404) {
            res.status(404).json({ nickname: 'Not found' });
        } else {
            res.status(500).json({ error: err.message });
        }
    }
The server gets authentication headers and contacts Autodesk's Design Automation API to check if your app has a nickname assigned. If no nickname exists, it returns "Not found" instead of an error.

🔍 Step 6: Handle Different Responses
If nickname exists: The server returns the nickname information
If no nickname: The server returns "Not found" (status 404)
If error: The server returns error details
This prevents the app from crashing if you haven't set up a nickname yet.

📱 Step 7: Show Results & Re-enable Buttons
    } catch (err) {
        logMessage(`--- ERROR ---<br/>${err.message}`);
    } finally {
        enableAllButtons(false);
    }
The browser receives the server's response and displays it in the output log. Then it makes all the buttons clickable again.

🎯 What This Accomplishes:
- Checks if your app has a Design Automation nickname
- Shows current nickname if one exists
- Shows "Not found" if no nickname is assigned
- Required to know if you need to set a nickname

📋 Summary for Non-Coders:
1. User presses "Get Nickname" button
2. All buttons become gray and output log clears
3. Browser sends GET request to server
4. Server gets authentication and contacts Autodesk
5. Autodesk checks if your app has a nickname assigned
6. Server returns either the nickname info or "Not found"
7. Browser shows the response in the output log
8. All buttons become clickable again

A nickname in Design Automation is like a unique username for your app. It's required before you can create AppBundles and Activities. This button just checks what nickname (if any) is currently assigned to your app credentials.
